# 霸凌报告系统重设计

## 目标
修改校园墙霸凌举报系统，实现更灵活的举报流程、数据唯一化、用户搜索集成、内容ID支持、管理后台处理弹窗和自动通知。

## 1. 霸凌报告机制更改

### 自我举报（self）
- 若用户已通过同学认证：直接进入表单，姓名自动填充（不可修改）
- 若用户未通过认证：弹窗显示两个选项
  - "前往认证"：跳转认证流程
  - "紧急情况"：允许手动填写姓名，进入表单
- 点击提交时：若未认证，先弹出同学认证弹窗
  - 认证窗口内的姓名必须与表单姓名一致（过滤空格后对比）
  - 认证提交后，霸凌报告一并提交

### 目击举报（witness）
- 不再要求同学认证，直接进入表单
- 不显示受害者姓名栏

## 2. 数据唯一化
- 在 `lib/uniqueId.js` 的 VALID_PREFIXES 中添加 `BULL`
- 后端使用 `generateId('BULL')` 生成报告 ID
- 写入 ID 日志

## 3. 涉事用户搜索
- 删除原有文本输入框
- 复用 `GET /api/users/search?q=xxx` 接口
- 前端展示分类搜索结果（匹配账号/昵称/UID/姓名）
- 点击用户后选择，点击"添加"按钮添加到列表
- 支持添加多个用户
- 数据格式：`involvedUsers: [{id, nickname, username}]`

## 4. 相关内容 ID
- "相关帖子ID" → "相关内容ID"
- 支持 POST/POCM/DISC/DICM/QAQU 等前缀 ID
- 用户输入 ID 后，前端自动匹配内容并显示
- 内容含图片则显示"(图片)"文字
- 支持添加多个内容 ID

## 5. 管理后台处理
- 详情弹窗显示完整表单、涉事用户详情、相关内容、举报人信息
- "确定处理" → 新弹窗选择封禁涉事用户 + 删除相关内容
- 封禁：调用 `POST /api/admin/users/:id/ban`
- 删除：调用相应删除接口，写入 `deleted_items` 表

## 6. 自动通知
- 确认霸凌 → T0 通知
- 非霸凌 → T0 通知
- 复用 `lib/penalty.js` 的 `emitUserNotice`
