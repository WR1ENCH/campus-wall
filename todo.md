# 测试清单

## 任务1：发帖类型选择改为彩色圆点

| 编号 | 测试项目 | 测试步骤 | 预期结果 |
|------|----------|----------|----------|
| 1.1 | 彩色圆点显示 | 打开发帖弹窗，观察类型选择区域 | 显示 5 个彩色圆点按钮（黄/粉/绿/橙/蓝），无文字和 SVG 图标 |
| 1.2 | 圆点选中状态 | 点击不同颜色的圆点 | 被点击的圆点放大（scale 1.2）并显示边框，其他圆点恢复半透明 |
| 1.3 | 默认选中 | 打开发帖弹窗，不操作类型选择 | 日常（黄色）圆点默认为选中态 |
| 1.4 | 帖子卡片颜色匹配 | 分别选择 5 种类型发帖 | 新发的帖子卡片背景色与所选类型圆点颜色匹配（日常→黄、表白→粉、树洞→绿、失物招领→橙、活动→蓝） |
| 1.5 | 卡片入场弹动动画 | 观察新帖子卡片出现过程 | 卡片入场时有弹动动画（colorPop：先放大 1.08 倍再回弹至正常大小） |
| 1.6 | 帖子类型文字显示 | 发帖后查看帖子卡片 | 卡片上仍显示类型文字（如「日常」「表白」等）和对应 SVG 图标 |
| 1.7 | 帖子详情颜色 | 点击帖子卡片打开详情弹窗 | 详情弹窗颜色与帖子类型匹配 |

## 任务2：全量访问记录

| 编号 | 测试项目 | 测试步骤 | 预期结果 |
|------|----------|----------|----------|
| 2.1 | 未登录访问记录 | 未登录状态下打开 index.html | admin.html 登录记录中出现一条 type='page_visit'、account='游客'、success=1 的记录 |
| 2.2 | 已登录访问记录 | 登录后打开 index.html | admin.html 登录记录中出现 type='page_visit'、account=用户昵称、success=1 的记录 |
| 2.3 | 访问记录含 IP/UA | 检查访问记录的详细信息 | 每条访问记录包含正确的 IP 地址和 User-Agent |
| 2.4 | 原有登录记录不受影响 | 执行一次用户登录 | 登录记录的 type='用户'、status='✅ 成功'/'❌ 失败' 仍正常显示 |
| 2.5 | 管理员登录记录不受影响 | 执行一次管理员登录 | 管理员登录记录的 type='管理员' 仍正常显示 |
| 2.6 | 留存期限 | 创建一条 100 天前的旧记录 | 该记录在下次写入时被自动删除，100 天内的记录保留 |
